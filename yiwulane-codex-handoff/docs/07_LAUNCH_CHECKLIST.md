# Owner Launch Checklist

## Brand and legal identity

- [ ] Confirm final domain.
- [ ] Confirm legal company name.
- [ ] Confirm registration country/region.
- [ ] Confirm public business address, if one will be shown.
- [ ] Confirm founder name and approved biography.
- [ ] Conduct trademark/domain review for YIWULANE.
- [ ] Have privacy, cookie, and terms pages reviewed by qualified counsel.

## Contact and conversion

- [ ] Set `BUSINESS_EMAIL`.
- [ ] Set public WhatsApp number.
- [ ] Set booking URL.
- [ ] Connect CRM/webhook.
- [ ] Test successful and failed lead delivery.
- [ ] Decide pilot fee and commercial terms.
- [ ] Approve lead-response SLA.

## Evidence

- [ ] Supply authentic founder photo.
- [ ] Supply Yiwu location/market images.
- [ ] Supply receiving/QC images.
- [ ] Supply packaging and labeling images.
- [ ] Supply warehouse pick/pack images.
- [ ] Approve captions and alt text.
- [ ] Approve first verified case study before showing numeric results.
- [ ] Approve real testimonials and written permission before publishing.

## Operations

- [ ] Define service boundaries and prohibited categories.
- [ ] Approve QC scope language.
- [ ] Approve claims about order synchronization/integrations.
- [ ] Approve countries and routes currently supported.
- [ ] Approve returns and exception-handling language.
- [ ] Confirm seller/importer responsibilities.
- [ ] Confirm insurance and liability wording.

## Analytics and SEO

- [ ] Configure GA4.
- [ ] Configure Microsoft Clarity.
- [ ] Verify analytics consent behavior.
- [ ] Verify Search Console.
- [ ] Submit sitemap.
- [ ] Replace staging canonicals with production URL.
- [ ] Review all resource drafts with a subject-matter expert.

## Final QA

- [ ] Test on current Chrome, Safari, Firefox, and Edge.
- [ ] Test iPhone and Android viewport behavior.
- [ ] Test keyboard-only navigation.
- [ ] Test form on slow and failed network.
- [ ] Test all email, phone, booking, and WhatsApp links.
- [ ] Confirm no placeholder or draft labels remain unintentionally.
- [ ] Confirm no unsupported metrics or fake proof.
- [ ] Confirm backup/rollback and DNS plan.
